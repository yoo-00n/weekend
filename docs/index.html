사용법

  • 페이지 우측 상단 Google 로그인 → 관리 기능 활성화

  • 사업지구 추가: 팀/이름/총필지/이번 주 완료 → [추가]

  • 주간 비교: 다음 주가 시작되면 [새 주 시작] 클릭 → 그 주부터 변경분이 **Δ(pp)**로 붉게 표시

  • 메모: LX/소관청 칸에 입력하면 onSnapshot으로 모두에게 실시간 반영

  • 샘플 데이터로 먼저 테스트해볼 수 있어요.

데이터 구조(파이어스토어)

projects 컬렉션의 문서 예:

json
{
  "name": "제주-서부1지구",
  "team": "A팀",
  "total": 140,
  "completed": 90,
  "completed_last_week": 75,
  "lxNotes": "도로 편입 3필지 보완필요",
  "authorityFeedback": "8/30 서류접수",
  "updatedAt": <server timestamp>
}

  • 공정률 = completed / total * 100

  • 주간 Δ(pp) = 이번 주 공정률 − 지난 주 공정률

  • [새 주 시작] 버튼이 모든 문서의 completed_last_week를 현재 completed로 일괄 복사합니다.

자주 묻는 포인트

  • 양쪽(기관)에서 동시에 편집? 네. Firestore의 onSnapshot으로 실시간 반영돼요.

  • 권한 분리가 필요하면: users/{uid}에 role을 저장하고 규칙에서 request.auth.uid로 분기하는 식으로 확장하세요.

  • 지표 커스터마이즈(예: 팀별 도넛, 누적 추이선)도 추가 가능—원하시면 이어서 확장해 드릴게요.

가장 흔한 원인 7가지 (빠른 점검)

  1. Firebase 설정 미입력

    • firebaseConfig가 YOUR_... 그대로면 코드가 “보기 전용(로그인 비활성)”으로 작동해요.
      → 콘솔에 Firebase 설정이 필요합니다... 경고가 찍힐 거예요. 내 프로젝트 값으로 교체하세요.

  2. 인증 제공자 미활성화

    • Firebase Console → Authentication → Sign-in method → Google “활성화” 필요.

  3. Authorized domains에 로컬 도메인 미추가

    • Authentication → Settings → Authorized domains에
      localhost, 127.0.0.1, 127.0.0.1:5500(포트 포함) 등을 추가하세요.
      (Live Server 포트를 쓰면 그 포트도 등록하는 게 안전합니다.)

  4. Firestore 생성/규칙 미설정

    • Firestore를 생성했고, 규칙이 아래처럼 “로그인 사용자 read/write 허용”인지 확인:

    js
    rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /projects/{doc} {
      allow read, write: if request.auth != null;
    }
    match /settings/{doc} {
      allow read, write: if request.auth != null;
    }
  }
}

  5. 팝업 차단

    • Google 로그인은 팝업을 씁니다. 브라우저 주소창 우측에 팝업 차단 아이콘이 뜨면 “허용”하세요.

  6. 프로젝트/컬렉션 비어 있음

    • 첫 화면이 비어 보일 수 있어요. 상단 로그인 → [샘플 데이터 추가] 버튼으로 확인해보세요.

    • 확인사항 메모는 settings/dashboard 문서를 자동 생성합니다(로그인 후 입력하면 저장).

  7. 콘솔 에러 무시

    • 개발자 도구(F12) → Console에서 에러 문구를 꼭 확인해 주세요.

    • auth/blocked-by-authorization → 도메인 미등록,
      permission-denied → 규칙 문제,
      app/invalid-api-key → config 오타가 대표적입니다.

ChatGPT의 말:

스크린샷에 보이는 줄 자체는 논리상 맞는 코드예요.
다만 VS Code에 표시된 저 작은 네모(�)가 실제 BOM 문자(U+FEFF) 로 붙어 있는 듯합니다. 이게 문자열 안이 아니라 코드에 “생짜”로 들어가면 구문 오류를 일으킬 수 있어요. 또, 일부 브라우저는 type:'text/csv;charset=utf-8;'의 세미콜론(;) 을 싫어합니다.

아래처럼 바꿔 붙이면 깔끔하게 해결됩니다. (BOM은 이스케이프 또는 바이트 배열로 넣고, MIME type의 세미콜론 제거)

js
function exportCSV(rows){
  const headers = ['team','name','total','completed','completed_last_week','progress_now','progress_prev','delta_pp','lxNotes','authorityFeedback'];
  const lines = [headers.join(',')];
  rows.forEach(r => {
    const total = Number(r.total)||0, comp=Number(r.completed)||0, last=Number(r.completed_last_week)||0;
    const pNow=pct(comp,total), pPrev=pct(last,total), d=(pNow-pPrev).toFixed(1);
    const vals = [r.team||'', r.name||'', total, comp, last, pNow.toFixed(1), pPrev.toFixed(1), d, quote(r.lxNotes||''), quote(r.authorityFeedback||'')];
    lines.push(vals.join(','));
  });

  // ✅ 방법 1: 유니코드 이스케이프로 BOM
  // const blob = new Blob(["\uFEFF" + lines.join('\r\n')], { type: 'text/csv;charset=utf-8' });

  // ✅ 방법 2: 바이트 배열로 BOM(가장 안전)
  const bom = new Uint8Array([0xEF, 0xBB, 0xBF]);
  const blob = new Blob([bom, lines.join('\r\n')], { type: 'text/csv;charset=utf-8' });

  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = 'projects.csv';
  document.body.appendChild(a);
  a.click();
  a.remove();
  setTimeout(()=> URL.revokeObjectURL(url), 0);
}

function quote(s){
  // replaceAll이 없다면 아래 한 줄을 사용: s.replace(/"/g, '""')
  const t = String(s).replaceAll('"','""');
  return `"${t}"`;
}

포인트

  • BOM 삽입: Excel에서 한글이 깨지지 않게 \uFEFF 혹은 바이트 배열(BOM)을 붙입니다.

  • 개행은 \r\n: 윈도우/엑셀 호환이 가장 좋습니다.

  • MIME type 끝의 세미콜론 제거: 'text/csv;charset=utf-8' 권장.

(로컬 Live Server와 GitHub Pages 둘 다 동일하게 동작합니다.)

0) 준비물

  • Google 계정

  • VS Code + Live Server

  • 내가 준 index.html (상단의 firebaseConfig 자리 존재)

1) Firebase 프로젝트 만들기

  1. Firebase 콘솔 → 프로젝트 추가.

  2. 프로젝트 이름 입력 → (Google Analytics는 선택).

  3. 생성 완료 후 좌측 프로젝트 설정(톱니바퀴) → 일반 탭 아래 내 앱 섹션에서 </> 웹 앱 추가 클릭.

  4. 웹 앱 닉네임만 적고 “Firebase 호스팅”은 체크 안 해도 됨 → 등록.

  5. 화면에 뜨는 config 스니펫에서 아래 부분을 복사해 둡니다:

    js
    const firebaseConfig = {
  apiKey: "....",
  authDomain: "YOURID.firebaseapp.com",
  projectId: "YOURID",
  storageBucket: "YOURID.appspot.com",
  messagingSenderId: "...",
  appId: "..."
};

2) index.html에 config 붙여넣기

  • 내 index.html 상단의 이 부분을 그대로 교체:

    js
    const firebaseConfig = {
  apiKey: "YOUR_API_KEY",
  authDomain: "YOUR_PROJECT_ID.firebaseapp.com",
  projectId: "YOUR_PROJECT_ID",
  storageBucket: "YOUR_PROJECT_ID.appspot.com",
  messagingSenderId: "YOUR_SENDER_ID",
  appId: "YOUR_APP_ID"
};

  • 저장하고 Live Server로 열면 최소한 “로그인 버튼”은 활성화되어야 정상 초기화입니다.

3) Authentication(로그인) 설정

  1. 콘솔 좌측 Build → Authentication.

  2. Sign-in method 탭 → Google 제공자 클릭 → 활성화.

  3. 화면 하단 Authorized domains에 다음을 추가:

    • localhost

    • 127.0.0.1

    • (나중에 GitHub Pages 쓸 거면) yourname.github.io 혹은 커스텀 도메인

  4. 프로젝트 지원 이메일이 필요하다는 경고가 보이면, 같은 화면 상단의 설정(톱니) → 일반에서 지원 이메일을 한 번 지정해 주세요.

에러 힌트

  • auth/unauthorized-domain → 위 “Authorized domains”에 도메인 추가 필요

  • 팝업이 안 뜨면 브라우저 팝업 차단 해제

4) Firestore 생성 & 규칙

  1. Build → Firestore Database → 데이터베이스 만들기.

  2. 지역 선택 → 프로덕션 모드 권장.

  3. 규칙(Rules) 탭에서 아래로 교체 → 게시:

    js
    rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // 대시보드 표 데이터
    match /projects/{doc} {
      allow read, write: if request.auth != null;
    }
    // 확인사항 메모
    match /settings/{doc} {
      allow read, write: if request.auth != null;
    }
  }
}

5) 로컬에서 로그인 → 동작 확인

  1. Live Server로 index.html 실행 → 우상단 Google 로그인.

  2. 로그인 성공하면 상단이 “OOO님 로그인”으로 바뀜 → [샘플 데이터 추가] 클릭.

  3. 표에 데이터가 뜨고, 수치/메모를 바꾸면 몇 초 내 저장되어야 정상입니다.

  4. Firestore 콘솔 데이터 탭을 보면 projects 컬렉션과 문서들이 생깁니다.
    확인사항을 입력하면 settings 컬렉션의 dashboard 문서에 checklist 필드가 생깁니다.

6) GitHub Pages로 배포(원하면)

  1. GitHub에 새 repo 만들고 index.html만 넣기.

  2. 루트에 .nojekyll (빈 파일) 추가 → Jekyll 빌드 이슈 방지.

  3. Repo → Settings → Pages → Branch: main / Folder: /(root) → Save.

  4. 발급된 URL(예: https://yourname.github.io/repo)을 Authentication → Authorized domains에 추가.

  5. 접속해서 로그인/편집이 로컬처럼 되는지 확인.

7) 자주 나는 에러와 해결

  • “Firebase 미설정 (보기 전용)”
    → firebaseConfig가 빈 값(YOUR_...) 상태. 실제 값으로 교체.

  • auth/unauthorized-domain
    → Authentication의 Authorized domains에 현재 접속 도메인 추가.

  • permission-denied (읽기/쓰기 실패)
    → Firestore 규칙이 로그인 허용으로 안 되어 있거나 게시를 안 눌렀음.

  • 로그인 창이 안 뜸
    → 브라우저 팝업 차단 해제.

  • 한글 CSV 깨짐
    → 내가 제공한 코드엔 BOM을 붙여서 엑셀 호환 OK(앞서 고쳐둔 버전 사용).

8) 데이터 구조 (참고)

  • 컬렉션 projects:
    name, team, total, completed, completed_last_week, lxNotes, authorityFeedback, updatedAt

  • 문서 settings/dashboard:
    checklist (대시보드 확인사항 메모)

“새 주 시작” 버튼을 누르면 모든 문서의 completed_last_week ← completed로 일괄 이월되어 Δ(pp) 계산이 바로 반영됩니다.
